## 🔍 상세 테스트 시나리오

### TC-01: 메인 랜딩 페이지 및 내비게이션
- **목적:** 사용자가 서비스의 가치를 즉시 파악하고 필요한 메뉴로 이동 가능한지 확인.
- **검증 절차:**
  1. `/` 경로 접속.
  2. `h1` 태그 내 "CJ PG로 결제 연동을 더 빠르고 안정적으로" 문구 확인.
  3. GNB의 'API' 클릭 시 `/api/payment`로 이동하는지 확인.
  4. 'API 시작하기' 버튼 클릭 시 결제 가이드로 리다이렉션되는지 확인.

### TC-02: 결제 API (Payment API) 문서
- **목적:** 결제 연동을 위한 필수 파라미터와 엔드포인트 정보의 정확성 검증.
- **검증 절차:**
  1. `/api/payment` 경로 접속.
  2. `POST https://api.cjone-pg.com/v2/payments/request` 엔드포인트 노출 확인.
  3. 필수 파라미터(`mid`, `orderId`, `amount`)가 테이블에 포함되어 있는지 확인.
  4. 우측 'Response JSON' 영역에 `success` 상태 코드 예시가 보이는지 확인.

### TC-03: 취소 API (Cancel API) 문서
- **목적:** 다양한 취소 시나리오(전체/부분)에 대한 가이드 제공 여부 검증.
- **검증 절차:**
  1. `/api/cancel` 경로 접속.
  2. '전체 취소' 및 '부분 취소' 설명 카드 렌더링 확인.
  3. `payment_id` 및 `cancel_amount` 파라미터의 필수 여부 확인.
  4. 하단 '주의사항' 섹션에서 은행 점검 시간 정보 노출 확인.

### TC-04: 결제 상태 조회 (Status API) 문서
- **목적:** 거래 상태 추적을 위한 API 명세 확인.
- **검증 절차:**
  1. `/api/status` 경로 접속.
  2. `GET /v1/payments/status/{tid}` 경로 확인.
  3. 상태 코드 테이블(200, 201, 404, 500)이 정상 출력되는지 확인.
  4. Webhook 알림 권장사항 섹션 존재 여부 확인.

### TC-05: API Playground (테스트 도구)
- **목적:** 개발자가 브라우저에서 직접 API를 테스트할 수 있는 UI 기능 검증.
- **검증 절차:**
  1. `/playground` 경로 접속.
  2. HTTP Method(POST/GET) 선택기 작동 확인.
  3. URL 입력 필드 및 JSON Body 편집기(코드 블록) 렌더링 확인.
  4. '요청 보내기' 버튼 클릭 시 Response 영역의 변화 확인.

### TC-06: 지원 센터 (Support)
- **목적:** 문의 내역 확인 및 신규 문의 등록 UI 검증.
- **검증 절차:**
  1. `/support` 경로 접속.
  2. '내 문의 리스트' 테이블에 샘플 데이터(답변완료, 처리중 등) 확인.
  3. '새 문의 등록' 폼에서 카테고리 선택 및 텍스트 입력 가능 여부 확인.

## ✅ 합격 기준 (Success Criteria)
1. 모든 시나리오에서 정의된 핵심 텍스트와 UI 요소가 브라우저에 렌더링되어야 함.
2. 메뉴 클릭 시 정의된 URL 경로로의 전환이 지연 없이 이루어져야 함.
3. Tailwind CSS 클래스 충돌로 인한 레이아웃 깨짐 현상이 없어야 함.
